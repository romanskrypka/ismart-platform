# Состояние Проекта (Memory Bank)

**Версия:** 1.0
**Дата:** 13.02.2026

## 1. Снимок Проекта

*   **Название:** iSmart Platform Documentation Overhaul
*   **Основная Цель:** Трансформировать iSmart Platform в вертикальную ИИ-компанию путем создания полного комплекта стратегической и технической документации, адаптированной к существующему PHP/Twig/JSON стеку.
*   **Текущая Фаза (согласно `DEVELOPMENT_PLAN.md`):** Фаза 1 - Фундамент и iSmart Start.
*   **Ключевая Активность:** Генерация основополагающих документов (`.cursorrules`, `VISION.md`, `USER_PROFILE.xml`, `DEVELOPMENT_PLAN.md` и др.).

## 2. Ключевые Решения и Выводы

1.  **Стратегический Поворот:** Принято решение о смене бизнес-модели — от продажи универсальной CMS к продаже специализированных «цифровых работников» для вертикальных рынков. Это зафиксировано в `VISION.md`.
2.  **Без Изменения Кода:** Текущая задача **не предполагает** модификации существующей кодовой базы iSmart Platform. Роль ИИ-агента — методолог и технический писатель, а не программист.
3.  **Адаптация Методологий:** Методологии PCAM, GRACE и FLEX, изначально ориентированные на Python/Django, адаптируются для стека PHP/Twig/JSON. Ключевые принципы адаптации зафиксированы в `.cursorrules` и `USER_PROFILE.xml`.
4.  **Гибридная Архитектура в Будущем:** `DEVELOPMENT_PLAN.md` описывает поэтапный переход к гибридной архитектуре, где текущая PHP-платформа будет служить фронтендом и API-шлюзом для нового ИИ-бэкенда на Python.
5.  **Нормализация Файлов:** В ходе анализа была выявлена проблема с именами файлов, содержащими скобки и пробелы. Файлы были скопированы в директорию `/home/ubuntu/docs_normalized/` для корректной обработки.

## 3. Краткий Обзор Архитектуры iSmart

*   **Ядро (`/core`):** Написано на PHP, управляет всем жизненным циклом запроса от роутинга до рендеринга.
*   **Данные (`/data/json`):** Весь контент, включая тексты, настройки и SEO-метаданные, хранится в структурированных JSON-файлах. Это ключевая особенность и точка для будущих интеграций.
*   **Шаблоны (`/templates`):** Используется шаблонизатор Twig с компонентной структурой (pages, sections, components).
*   **Фронтенд (`/assets`):** Собирается с помощью Webpack 5, используется нативный JavaScript (ES6) и PostCSS.
*   **Точка входа (`index.php`):** Единая точка входа, которая инициализирует приложение.

## 4. Адаптация Методологий: Кратко

*   **GRACE для PHP:**
    *   `MODULE_CONTRACT`: В виде отдельных `*.contract.md` файлов.
    *   `BLOCK`: Через структурированные PHP-комментарии (`// BLOCK:`).
    *   `LOGGING`: Через существующий класс `Logger`.
*   **PCAM для Документации:**
    *   `Purpose`: `VISION.md`.
    *   `Context`: `USER_PROFILE.xml`, `.cursorrules`, код.
    *   `Action` & `Microsteps`: План выполнения текущей задачи.
