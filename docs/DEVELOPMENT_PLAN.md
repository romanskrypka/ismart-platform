# План Развития Платформы iSmart

**Версия:** 1.0
**Дата:** 13.02.2026

## 1. Введение

Этот документ описывает стратегический план развития платформы iSmart, направленный на её трансформацию из модульной веб-платформы в полноценную **вертикальную ИИ-компанию**. План основан на видении, изложенном в `VISION.md`, и предполагает поэтапную эволюцию продукта, архитектуры и технологий.

**Основная цель:** Переход от продажи технологии (CMS) к продаже измеримого бизнес-результата через предоставление специализированных, обучаемых ИИ-агентов («цифровых работников»).

## 2. Фазы Развития

План разделен на три последовательные фазы, каждая из которых имеет свои цели, ключевые инициативы и метрики успеха.

### Фаза 1: Фундамент и iSmart Start (Сроки: 0-6 месяцев)

**Цель:** Укрепить текущую PHP-платформу в качестве предложения **«iSmart Start»** и заложить основу для будущей технологической эволюции.

| Направление | Ключевые инициативы |
| :--- | :--- |
| **Методология** | 1.  **Полное документирование:** Описать существующую кодовую базу PHP с использованием адаптированной методологии **GRACE**. Создать `*.contract.md` для всех ключевых классов в `/core`.<br>2.  **Внедрение конвенций:** Разработать и внедрить `CONVENTIONS.md`, описывающие правила работы с кодом, JSON и шаблонами. |
| **Продукт** | 1.  **Пилотные проекты:** Запустить 2-3 пилотных проекта для целевых вертикалей (например, автодилеры, недвижимость) на текущей PHP-платформе.<br>2.  **«Легкое» ИИ-внедрение:** Интегрировать в пилотные проекты сторонний RAG-чат (через API) для демонстрации ИИ-возможностей и сбора обратной связи. |
| **Технология** | 1.  **Подготовка к API:** Провести ревизию ключевых функций (отправка форм, отдача контента) и спроектировать внутренний API для будущего использования.<br>2.  **Аналитические скрипты:** Разработать отдельные Python-скрипты для анализа логов и данных, собранных с пилотных проектов. |

**Технологический стек:** PHP 8.1+, Twig, JSON, Vanilla JS, Python (для аналитики).

**Метрики успеха:**

*   Запущено не менее 2 пилотных проектов.
*   Покрыто 80% классов в директории `/core` файлами `.contract.md`.
*   Создана и утверждена первая версия спецификации внутреннего API.

### Фаза 2: Цифровые Работники и Гибридная Архитектура (Сроки: 6-18 месяцев)

**Цель:** Запустить первого «цифрового работника» и перейти к гибридной архитектуре, где PHP и Python сосуществуют и взаимодействуют.

| Направление | Ключевые инициативы |
| :--- | :--- |
| **Продукт** | 1.  **Первый «цифровой работник»:** Разработать первого коммерческого ИИ-агента (например, «ИИ-ассистент по лидам» для автодилера), который автоматизирует конкретную бизнес-роль.<br>2.  **Модель подписки:** Перевести первого клиента на модель SaaS-подписки за использование «цифрового работника». |
| **Технология** | 1.  **ИИ-сервис на Python:** Создать новый бэкенд-сервис на **Python/Django/FastAPI** для размещения и управления ИИ-агентами.<br>2.  **API-шлюз:** Реализовать API-шлюз, который будет маршрутизировать запросы: контентные — на PHP-бэкенд, ИИ-задачи — на новый Python-сервис.<br>3.  **Интеграции:** Реализовать в Python-сервисе интеграцию как минимум с одной популярной CRM-системой (например, Bitrix24, amoCRM). |

**Технологический стек:** PHP (веб-контент), **Python/Django** (ИИ-бэкенд), PostgreSQL, REST API.

**Метрики успеха:**

*   Первый клиент платит по SaaS-подписке за «цифрового работника».
*   Успешно работает интеграция между PHP-фронтендом и Python-бэкендом.
*   Реализована и используется интеграция с одной CRM/ERP системой.

### Фаза 3: Обучающаяся Платформа и Полная Миграция (Сроки: 18-36 месяцев)

**Цель:** Создать самообучающуюся экосистему и завершить миграцию на единую технологическую платформу.

| Направление | Ключевые инициативы |
| :--- | :--- |
| **Продукт** | 1.  **Замыкание петли обучения:** Реализовать контур обратной связи (RLHF), где результаты работы агентов используются для их дообучения и улучшения.<br>2.  **Партнерская экосистема:** Разработать фреймворк и документацию для сторонних разработчиков, позволяющие им создавать и монетизировать своих «цифровых работников» на платформе iSmart. |
| **Технология** | 1.  **Полная миграция:** Постепенно перенести всю логику рендеринга и управления контентом с PHP-платформы на Python-бэкенд.<br>2.  **Хранилище данных:** Внедрить централизованное хранилище данных (Data Warehouse) для сбора и анализа метрик со всех клиентских проектов.<br>3.  **Отказ от PHP:** Вывести из эксплуатации легаси-платформу на PHP. |

**Технологический стек:** **Python/Django**, продвинутые ML/RL фреймворки (например, LangGraph), решения для DWH (например, ClickHouse).

**Метрики успеха:**

*   Зафиксировано и продемонстрировано клиенту измеримое улучшение бизнес-метрик за счет самообучения ИИ-агентов.
*   90% веб-трафика и бизнес-логики обслуживается новой платформой на Python.
*   Первый сторонний «цифровой работник» развернут на платформе. |
