# МОДУЛЬ 0: Рабочие шаблоны и чек-листы

## Введение

Эти шаблоны и фреймворки предназначены для того, чтобы помочь вам структурировать вашу работу по созданию Personal OS. Используйте их как отправную точку для планирования, проектирования и оценки ваших персональных AI-решений.

---

### Фреймворк 1: Канвас проектирования Personal OS

Используйте эту таблицу, чтобы спроектировать архитектуру вашей персональной операционной системы. Заполните каждую секцию, чтобы получить целостное видение вашей будущей системы.

| Компонент | Описание | Ваши заметки |
| :--- | :--- | :--- |
| **1. Цели и Задачи** | Какие ключевые цели вы хотите достичь с помощью Personal OS? Какие рутинные или сложные задачи вы хотите автоматизировать? (Например: "Сократить время на написание отчетов на 50%", "Автоматизировать мониторинг новостей по моей отрасли") | |
| **2. Ключевые рабочие процессы** | Опишите 2-3 основных рабочих процесса, которые вы хотите улучшить. (Например: "Еженедельная подготовка аналитики", "Сортировка и ответ на входящие email", "Планирование контента для блога") | |
| **3. Источники данных (для RAG)** | Какую информацию должен знать ваш ассистент? К каким документам, базам знаний, сайтам, заметкам ему нужен доступ? (Например: "Мой архив рабочих документов в Google Drive", "Внутренняя wiki компании", "Мои личные заметки в Obsidian") | |
| **4. Необходимые навыки (Skills)** | Какие конкретные навыки (промпты/скрипты) нужно разработать для реализации ваших рабочих процессов? (Например: "summarize_report", "draft_email_reply", "generate_social_media_post") | |
| **5. Инструментарий** | Какие инструменты вы будете использовать? (IDE, языковые модели, API, векторные базы данных и т.д.) | |
| **6. Критерии успеха** | Как вы поймете, что ваша Personal OS эффективна? Какие метрики вы будете отслеживать? (Например: "Сэкономленное время в часах/неделю", "Количество автоматически выполненных задач", "Снижение когнитивной нагрузки") | |

### Фреймворк 2: Шаблон для проектирования навыка (Skill)

Используйте этот шаблон для детальной проработки каждого нового навыка перед его реализацией. Это поможет создавать более надежные и предсказуемые AI-функции.

| Элемент промпта | Описание | Реализация для вашего навыка |
| :--- | :--- | :--- |
| **Название навыка** | Краткое, понятное имя (например, `generate_meeting_agenda`). | |
| **Цель навыка** | Что этот навык должен делать? | |
| **Роль (Role)** | Кем должен "притвориться" AI? | |
| **Задача (Task)** | Четкая и однозначная постановка задачи. | |
| **Контекст (Context)** | Какая информация будет передаваться в промпт динамически? (переменные `{...}`) | |
| **Пошаговые инструкции** | Детальный алгоритм действий для AI. | |
| **Формат вывода** | В каком виде должен быть результат? (JSON, Markdown, и т.д.) | |
| **Ограничения и правила** | Чего AI не должен делать? Какие рамки нужно соблюдать? | |
| **Примеры (Few-shot)** | (Опционально) 1-2 примера пары "вход -> идеальный выход" для более точной настройки. | |
