# МОДУЛЬ 0: Практические задания

## Введение

Этот набор практических заданий поможет вам применить на практике концепции, изученные в лекции, и сделать первые шаги к созданию вашей персональной операционной системы (Personal OS). Выполняйте задания последовательно, так как они построены по принципу "от простого к сложному".

---

### Задание 1: Настройка рабочей среды (Базовый уровень)

**Цель:** Установить и настроить IDE Cursor, чтобы подготовить плацдарм для дальнейшей работы.

**Тайминг:** 25-30 минут.

**Пошаговая инструкция:**

1.  **Скачайте и установите Cursor:**
    *   Перейдите на официальный сайт [cursor.sh](https://cursor.sh/).
    *   Скачайте версию для вашей операционной системы (Windows, macOS, Linux).
    *   Установите приложение, следуя инструкциям инсталлятора.

2.  **Первый запуск и настройка AI:**
    *   Откройте Cursor. При первом запуске вам будет предложено настроить подключение к языковым моделям.
    *   Вы можете использовать встроенные модели Cursor (которые работают "из коробки", но с определенными лимитами) или подключить свой соб-ственный API-ключ от OpenAI (GPT-4, GPT-3.5) или Anthropic (Claude).
    *   Для начала рекомендуется использовать встроенные возможности. Нажмите `Ctrl+K` (или `Cmd+K` на macOS) чтобы открыть чат с AI и задайте простой вопрос, например, "Привет! Что ты умеешь?". Убедитесь, что вы получаете ответ.

3.  **Создание проекта Personal OS:**
    *   Создайте на вашем компьютере новую папку с названием `personal-os`.
    *   В Cursor откройте эту папку (`File -> Open Folder...`).
    *   Внутри папки создайте подпапку `skills`.
    *   Это будет корневая директория вашего проекта.

4.  **Знакомство с интерфейсом:**
    *   Попробуйте основные функции Cursor:
        *   **Chat with your code:** Выделите фрагмент кода (или просто пустую строку в файле) и нажмите `Ctrl+K`. Задайте вопрос о коде или попросите сгенерировать новый.
        *   **Edit with AI:** Выделите код, нажмите `Ctrl+K` и опишите, какие изменения вы хотите внести. Например, "перепиши эту функцию, чтобы она была более читаемой".

**Ожидаемый результат:**

*   На вашем компьютере установлена и работает IDE Cursor.
*   Создана структура папок для проекта `personal-os`.
*   Вы успешно отправили как минимум один запрос к AI через интерфейс Cursor и получили ответ.

**Критерии оценки:**

*   **Выполнено (3/3):** Все пункты ожидаемого результата достигнуты.
*   **Частично выполнено (2/3):** Cursor установлен, но возникли проблемы с настройкой AI или созданием проекта.
*   **Не выполнено (1/3):** Не удалось установить или запустить Cursor.

### Задание 2: Создание первого навыка (Промпт-инжиниринг)

**Цель:** Разработать и протестировать простой, но полезный навык, используя только промпт-инжиниринг.

**Тайминг:** 30-40 минут.

**Пошаговая инструкция:**

1.  **Определите задачу:** Мы создадим навык для генерации профессиональных, но вежливых отказов на предложения о встрече, которые не соответствуют вашим текущим приоритетам.

2.  **Разработайте промпт:**
    *   В папке `skills` вашего проекта создайте новый файл `decline_meeting.md`.
    *   В этом файле напишите промпт, который будет служить основой вашего навыка. Используйте структуру, описанную в лекции (роль, задача, контекст, формат).
    *   **Пример промпта (вы можете его улучшить):**
        ```
        **Роль:** Ты — мой персональный ассистент, мастер деловой коммуникации.

        **Задача:** Напиши вежливый, но твердый отказ на предложение о встрече. Причина отказа — несоответствие моим текущим стратегическим приоритетам.

        **Контекст:**
        - **Мои текущие приоритеты:** {my_priorities}
        - **Тема предложенной встречи:** {meeting_topic}
        - **Кто предложил:** {requester_name}

        **Инструкции:**
        1.  Поблагодари за предложение.
        2.  Кратко и без лишних деталей объясни, что в данный момент я сфокусирован(а) на других задачах.
        3.  Не оставляй возможности для дальнейшего обсуждения этой темы сейчас, но будь открыт(а) к контактам в будущем, если их цели изменятся.
        4.  Стиль: профессиональный, лаконичный, дружелюбный.
        5.  Язык: русский.

        **Формат вывода:** Готовый текст письма.
        ```

3.  **Протестируйте навык в Cursor:**
    *   Откройте чат AI (`Ctrl+K`).
    *   Скопируйте ваш промпт из файла `decline_meeting.md` в чат.
    *   Замените переменные в фигурных скобках `{...}` на конкретные примеры. Например:
        *   `{my_priorities}`: "Запуск нового продукта в Q3 и оптимизация внутренних процессов."
        *   `{meeting_topic}`: "Интеграция с новой CRM-системой."
        *   `{requester_name}`: "Иван Петров."
    *   Отправьте промпт и оцените результат.

4.  **Итерация и улучшение:**
    *   Если результат вас не устраивает, доработайте промпт. Возможно, стоит добавить больше деталей в инструкции или изменить формулировки. 
    *   Попробуйте несколько разных вариантов входных данных, чтобы убедиться, что навык работает стабильно.
    *   Сохраните финальную, лучшую версию промпта в файле `decline_meeting.md`.

**Ожидаемый результат:**

*   Создан файл `skills/decline_meeting.md`, содержащий проработанный промпт.
*   Навык протестирован в Cursor как минимум на двух разных примерах входных данных.
*   Сгенерированный текст письма соответствует заданным критериям (вежливый, профессиональный, ясный отказ).

**Критерии оценки:**

*   **Выполнено (3/3):** Создан и протестирован качественный промпт, результат генерации соответствует задаче.
*   **Частично выполнено (2/3):** Промпт создан, но требует доработки, или результат не всегда соответствует ожиданиям.
*   **Не выполнено (1/3):** Промпт не создан или не работает.

### Задание 3: Автоматизация навыка (Продвинутый уровень)

**Цель:** Превратить промпт-навык из Задания 2 в автоматизированный скрипт, который можно вызывать из командной строки.

**Тайминг:** 45-60 минут.

**Пошаговая инструкция:**

1.  **Подготовка:**
    *   Убедитесь, что у вас на компьютере установлен Python (обычно он уже есть на macOS и Linux, для Windows его нужно установить с [python.org](https://python.org/)).
    *   Вам понадобится API-ключ от OpenAI или Anthropic. Если у вас его нет, вы можете зарегистрироваться на их сайтах и получить его. **Внимание:** использование API может быть платным.
    *   Установите необходимые библиотеки. Откройте терминал в Cursor (`View -> Terminal`) и выполните команды:
        ```bash
        pip install openai
        pip install anthropic
        ```

2.  **Создание скрипта:**
    *   В папке `skills` создайте файл `decline.py`.
    *   Напишите Python-скрипт, который будет:
        *   Принимать аргументы командной строки (тему встречи и имя предложившего).
        *   Считывать шаблон промпта из файла `decline_meeting.md`.
        *   Форматировать промпт, подставляя в него полученные аргументы и ваши заранее заданные приоритеты.
        *   Отправлять запрос к API выбранной языковой модели.
        *   Выводить результат в консоль.

3.  **Настройка безопасности:**
    *   **НИКОГДА** не храните API-ключ прямо в коде. Используйте переменные окружения. В терминале перед запуском скрипта выполните команду:
        ```bash
        export OPENAI_API_KEY='ваш_ключ'
        # или
        export ANTHROPIC_API_KEY='ваш_ключ'
        ```

4.  **Запуск и тестирование:**
    *   Откройте терминал в Cursor.
    *   Перейдите в директорию `skills` (`cd skills`).
    *   Запустите скрипт с параметрами:
        ```bash
        python decline.py "Интеграция с новой CRM" "Иван Петров"
        ```
    *   Убедитесь, что скрипт отрабатывает корректно и выводит в консоль сгенерированный текст письма.

**Ожидаемый результат:**

*   Создан и работает скрипт `skills/decline.py`.
*   Скрипт успешно принимает аргументы командной строки, взаимодействует с API языковой модели и возвращает результат.
*   API-ключ не хранится в коде, а используется через переменные окружения.

**Критерии оценки:**

*   **Выполнено (3/3):** Скрипт полностью функционален и соответствует всем требованиям, включая безопасность.
*   **Частично выполнено (2/3):** Скрипт работает, но есть недочеты (например, ключ в коде) или ошибки при обработке некоторых входных данных.
*   **Не выполнено (1/3):** Скрипт не работает или не создан.
