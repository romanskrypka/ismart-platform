# Рабочие шаблоны и чек-листы

## Введение

Этот документ содержит набор практических шаблонов, чек-листов и фреймворков, которые помогут вам в проектировании и реализации систем сквозной автоматизации. Используйте их как отправную точку для анализа ваших собственных бизнес-процессов и создания мультиагентных workflow.

## Шаблон 2: Фреймворк Classify → Route → Execute → Feedback (CREF)

Этот шаблон поможет вам детально проработать весь жизненный цикл автоматизированной задачи, применяя фреймворк CREF.

**Задача:** [Название конкретной задачи, например, "Обработать запрос на возврат товара"]

| Этап | Описание действий | Ответственный агент(ы) | Входные данные | Выходные данные (результат) | 
| :--- | :--- | :--- | :--- | :--- | 
| **Classify** | *Анализ письма от клиента для подтверждения намерения возврата и извлечения номера заказа.* | *Агент-Диспетчер* | *Текст письма* | *{ "intent": "return_request", "order_id": "XYZ-123" }* | 
| **Route** | *На основе намерения "return_request" направить задачу Агенту по возвратам.* | *Оркестратор* | *Выходные данные этапа Classify* | *Задача, переданная Агенту по возвратам* | 
| **Execute** | *1. Проверить статус заказа в CRM.<br>2. Проверить соответствие условиям возврата.<br>3. Создать заявку на возврат в системе.* | *Агент по возвратам* | *Номер заказа* | *{ "return_status": "approved", "return_id": "R-456" }* | 
| **Feedback** | *1. Отправить клиенту подтверждение о создании заявки.<br>2. Сохранить результат (успешный возврат) в базу знаний для анализа.* | *Агент по коммуникациям, Агент-Логгер* | *Результат этапа Execute* | *Отправленное письмо, запись в лог-файле* |

## Чек-лист 1: Самопроверка при проектировании сквозной автоматизации

Используйте этот чек-лист, чтобы убедиться, что вы учли все ключевые аспекты при проектировании вашей системы.

### Фаза 1: Анализ и планирование
- [ ] Четко определен бизнес-процесс для автоматизации.
- [ ] Измерены текущие метрики процесса (время, стоимость, ошибки).
- [ ] Определены четкие цели автоматизации (KPI).
- [ ] Проведена оценка целесообразности (ROI).

### Фаза 2: Проектирование архитектуры
- [ ] Определен состав «команды» AI-агентов.
- [ ] Роли и обязанности каждого агента четко разграничены.
- [ ] Выбран подходящий шаблон оркестрации (Sequential, Parallel, Group Chat и т.д.).
- [ ] Определена роль Оркестратора.
- [ ] Спроектирован протокол и формат межагентной коммуникации (например, JSON-сообщения).
- [ ] Продуман механизм обработки ошибок и исключений.

### Фаза 3: Реализация фреймворка CREF
- [ ] **Classify:** Определены все типы входящих запросов и их атрибуты.
- [ ] **Route:** Прописаны все правила маршрутизации задач.
- [ ] **Execute:** Для каждого агента-исполнителя определен полный набор его действий и необходимых инструментов (API, БД).
- [ ] **Feedback:** Спроектирован механизм сбора обратной связи (внутренней и внешней).
- [ ] **Feedback:** Продуман процесс использования обратной связи для улучшения системы (цикл обучения).

### Фаза 4: Безопасность и мониторинг
- [ ] Продуманы вопросы безопасности (управление доступами агентов).
- [ ] Определены метрики для мониторинга производительности системы.
- [ ] Настроен механизм логирования всех действий агентов.
- [ ] Предусмотрен «аварийный выключатель» или механизм для плавной передачи управления человеку.

## Матрица 1: Оценка и приоритизация процессов для автоматизации

Эта матрица поможет вам выбрать, какие бизнес-процессы стоит автоматизировать в первую очередь. Оцените каждый процесс-кандидат по двум осям: **Потенциальная выгода** и **Сложность реализации**. 

| Процесс-кандидат | Потенциальная выгода (1-10) | Сложность реализации (1-10) | Приоритет (Выгода / Сложность) | Примечания | 
| :--- | :--- | :--- | :--- | :--- | 
| *Обработка счетов-фактур* | *8* | *3* | *2.67* | *Высокая повторяемость, структурированные данные* | 
| *Онбординг клиентов* | *9* | *7* | *1.28* | *Требует интеграции с множеством систем* | 
| *Модерация отзывов* | *6* | *5* | *1.20* | *Требует качественной NLP-модели* | 
| *Ежедневная рассылка отчетов* | *5* | *2* | *2.50* | *Простая задача, но низкая выгода* | 
| | | | | | 

**Как использовать:**
1.  Составьте список процессов-кандидатов на автоматизацию.
2.  Оцените каждый по 10-балльной шкале:
    *   **Потенциальная выгода:** Насколько сильно автоматизация сократит расходы, увеличит скорость, повысит удовлетворенность клиентов?
    *   **Сложность реализации:** Насколько сложны техническая реализация, интеграция, требуемые AI-модели?
3.  Рассчитайте приоритет. Процессы с самым высоким коэффициентом приоритета — ваши лучшие кандидаты для старта.
