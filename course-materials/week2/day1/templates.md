## 2. Шаблон для описания графа в JSON-LD (для Задания 3)

Используйте этот шаблон для формализации небольшого фрагмента вашего графа знаний. Замените примеры на ваши собственные сущности и связи.

**Контекст примера:** *Проект "Запуск нового сайта"*, которым руководит *сотрудник "Иванов Иван"*, и в рамках которого есть *задача "Нарисовать макеты"*. 

```json
[
  {
    "@context": "https://schema.org",
    "@type": "Project",
    "name": "Запуск нового сайта",
    "description": "Проект по разработке и запуску нового корпоративного веб-сайта.",
    "startDate": "2024-09-01",
    "endDate": "2024-12-31",
    "projectManager": {
      "@type": "Person",
      "name": "Иванов Иван",
      "email": "i.ivanov@example.com",
      "jobTitle": "Менеджер проектов"
    },
    "hasTask": [
      {
        "@type": "Task",
        "name": "Нарисовать макеты",
        "status": "В работе",
        "assignee": {
          "@type": "Person",
          "name": "Петрова Анна",
          "jobTitle": "UI/UX Дизайнер"
        }
      }
    ]
  }
]
```

## 3. Чек-лист для самопроверки

Используйте этот чек-лист, чтобы убедиться, что вы качественно выполнили все задания.

### Задание 1: Моделирование сущностей

- [ ] Я выбрал(а) конкретную и понятную предметную область для анализа.
- [ ] Я описал(а) как минимум 3-5 ключевых сущностей.
- [ ] Среди моих сущностей есть представители как минимум 3 разных типов (Роли, Процессы, Данные, Метрики).
- [ ] Для каждой сущности я указал(а) 3-5 релевантных атрибута.
- [ ] Все описания в таблице четкие и не допускают двойного толкования.

### Задание 2: Построение Контекстного графа

- [ ] Мой граф содержит 5-7 сущностей из реальной практики моего бизнеса.
- [ ] Все сущности на графе соединены между собой.
- [ ] Связи имеют понятные и осмысленные названия (например, «отвечает за», «использует»).
- [ ] Связи направленные (показаны стрелками).
- [ ] В графе есть не только прямые связи от центральной сущности, но и связи "второго уровня".
- [ ] Граф визуально понятен и хорошо структурирован.

### Задание 3: Формализация онтологии

- [ ] Я выбрал(а) для описания 2-3 связанные сущности из моего графа.
- [ ] Я использовал(а) синтаксис JSON-LD.
- [ ] Код является валидным JSON.
- [ ] Структура JSON-объектов (ключи, вложенность) соответствует связям, которые я нарисовал(а) в графе.
- [ ] В JSON я указал(а) не только названия сущностей, но и их ключевые атрибуты.
