### 2. События, подлежащие аудиту

| ID События | Описание события | Категория (Security, Performance, Business) |
| :--- | :--- | :--- |
| `EVT-001` | Агент получил новый запрос | `Business` |
| `EVT-002` | Агент начал обработку запроса | `Performance` |
| `EVT-003` | Агент успешно аутентифицировался в системе X | `Security` |
| `EVT-004` | Ошибка аутентификации в системе X | `Security` |
| `EVT-005` | Агент выполнил операцию чтения данных Y | `Security` |
| `EVT-006` | Агент выполнил операцию записи данных Z | `Security` |
| `EVT-007` | Агент принял решение A на основе данных B | `Business` |
| `EVT-008` | Агент завершил обработку запроса | `Business` |
| `...` | | |

### 3. Структура записи в журнале аудита (Log Schema)

```json
{
  "timestamp": "(string, ISO 8601)",
  "agent_id": "(string)",
  "agent_role": "(string)",
  "event_id": "(string, из таблицы выше)",
  "event_source": "(string, IP или имя хоста)",
  "action": {
    "name": "(string, напр. READ, WRITE, EXECUTE)",
    "target_resource": "(string, напр. /api/v1/users)",
    "status": "(string, SUCCESS или FAILURE)",
    "error_message": "(string, если status=FAILURE)"
  },
  "context": {
    "request_id": "(string, для сквозной трассировки)",
    "user_id": "(string, если действие инициировано пользователем)",
    "input_data_hash": "(string, хэш входных данных для проверки целостности)",
    "decision_rationale": "(string, краткое объяснение, почему агент принял такое решение)"
  }
}
```

### 4. Правила проактивного мониторинга и оповещений (Alerting Rules)

| ID Правила | Описание правила | Уровень критичности (Info, Warning, Critical) | Получатель оповещения |
| :--- | :--- | :--- | :--- |
| `RULE-01` | Более 5 событий `EVT-004` (Ошибка аутентификации) от одного агента за 1 минуту. | `Critical` | Security Team, Agent Owner |
| `RULE-02` | Агент пытается выполнить действие, не разрешенное его ролью (на основе матрицы RBAC). | `Critical` | Security Team |
| `RULE-03` | Время обработки запроса (от `EVT-002` до `EVT-008`) превысило 60 секунд. | `Warning` | Agent Owner |
| `...` | | | |
