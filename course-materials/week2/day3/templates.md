
# Рабочие шаблоны и чек-листы: Playbook Внедрения

## Введение

Этот документ содержит рабочие материалы для воркшопа «Playbook Внедрения». Используйте эти шаблоны и чек-листы для выполнения практических заданий и для дальнейшей работы над вашими AI-инициативами. Они помогут структурировать ваши мысли и обеспечат системный подход к внедрению AI-агентов.

---

## Шаблон 1. Алгоритм описания процесса (Фреймворк ЦКДО)

**Инструкция:** Используйте этот шаблон для создания четкого «технического задания» для вашего AI-агента. Заполните каждый из четырех блоков.

### Название AI-инициативы: ______________________________

| Компонент | Описание |
| :--- | :--- |
| **Цель (Goal)** | *Что конкретно должен сделать агент? Какой конечный, измеримый результат ожидается?* <br><br> **Пример:** Классифицировать все входящие письма в папке support@company.com на три категории: «Технический вопрос», «Вопрос по оплате», «Другое» в течение 5 минут после получения. |
| **Контекст (Context)** | *Какую фоновую информацию должен знать агент? К каким данным или системам у него есть доступ? Какую роль он исполняет?* <br><br> **Пример:** Ты — агент поддержки первой линии. У тебя есть доступ к базе знаний (knowledge_base.pdf) и истории обращений клиента в CRM. Клиенты могут быть раздражены. Твой тон должен быть всегда вежливым и эмпатичным. |
| **Действия (Actions)** | *Опишите пошаговый алгоритм работы агента. Что он делает в первую очередь, что во вторую и т.д.?* <br><br> **Пример:** <br> 1. Получи новое письмо из папки support@company.com. <br> 2. Проанализируй текст письма. <br> 3. Если в письме есть слова „ошибка“, „не работает“, „баг“ — присвой категорию „Технический вопрос“. <br> 4. ... |
| **Ограничения (Constraints)** | *Чего агент категорически не должен делать? Какие есть «красные линии»?* <br><br> **Пример:** Никогда не обещай клиенту точные сроки решения технической проблемы. Никогда не предлагай скидку без согласования. Не используй в ответе более 200 слов. |
