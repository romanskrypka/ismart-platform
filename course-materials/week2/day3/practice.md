
# Практические задания: Playbook Внедрения

## Общая информация

Этот набор практических заданий предназначен для закрепления материала воркшопа «Playbook Внедрения». Цель — пройти полный цикл от формулировки гипотезы до создания плана реагирования на ошибки для вашей собственной AI-инициативы. Работа может выполняться индивидуально или в группах по 2-3 человека.

---

## Задание 1. Мозговой штурм и выбор гипотезы (Базовый уровень)

**Задача:** Идентифицировать и выбрать одну наиболее перспективную идею для внедрения AI-агента в вашей реальной рабочей деятельности.

**Инструкция:**

1.  **Проведите мозговой штурм (15 минут):**
    *   Выпишите 5-7 рутинных, трудоемких или неэффективных процессов в вашей ежедневной работе или работе вашего отдела.
    *   Для каждого процесса кратко опишите, в чем заключается основная проблема (например, «долго», «частые ошибки», «требует много ручного труда»).
2.  **Оцените потенциал для AI (10 минут):**
    *   Напротив каждого процесса поставьте оценку от 1 до 5, где 1 — «AI здесь вряд ли поможет», а 5 — «идеальная задача для AI-агента».
    *   Руководствуйтесь критериями: повторяемость, работа с текстом/данными, наличие четких правил.
3.  **Выберите одну гипотезу (5 минут):**
    *   Выберите процесс с самой высокой оценкой. Сформулируйте для него гипотезу по шаблону: «Если мы внедрим AI-агента для [описание процесса], то мы достигнем [ожидаемый результат]».

**Критерии оценки:**

-   Выбрана релевантная для вашей работы проблема.
-   Гипотеза сформулирована четко и по шаблону.
-   Ожидаемый результат измерим (например, «сократим время на...», «увеличим количество...», «снизим число ошибок на...»).

**Ожидаемые результаты:**

-   Список из 5-7 проблемных процессов.
-   Одна четко сформулированная гипотеза для дальнейшей работы.

**Тайминг:** 30 минут.

## Задание 2. Экспресс-тестирование гипотезы (Продвинутый уровень)

**Задача:** Провести быструю проверку выбранной гипотезы, используя чек-лист «Time-to-Value < 2 часов».

**Инструкция:**

1.  **Заполните чек-лист (Шаги 1-4 из лекции):**
    *   **Формулировка (15 минут):** Детально распишите проблему, гипотезу и ожидаемую ценность.
    *   **Подготовка MVP (60 минут):**
        *   Выберите доступный вам AI-инструмент (например, веб-версию ChatGPT, Claude, Gemini или корпоративный аналог).
        *   Подготовьте минимальный набор данных для теста (например, один документ, одно письмо, одна таблица).
        *   Напишите упрощенный промпт для AI-агента, используя фреймворк ЦКДО (Цель-Контекст-Действия-Ограничения).
    *   **Эксперимент (30 минут):**
        *   Самостоятельно или с коллегой выполните 3-5 тестовых запросов к вашему «прото-агенту».
        *   Зафиксируйте результаты: время выполнения и субъективную оценку качества ответа.
    *   **Анализ (15 минут):**
        *   Сравните полученные результаты с текущим (ручным) способом выполнения задачи.
        *   Примите решение: «Go» (гипотеза имеет потенциал) или «No-Go» (идея нежизнеспособна в текущем виде).

**Критерии оценки:**

-   Заполнены все пункты чек-листа.
-   Промпт для агента составлен по фреймворку ЦКДО.
-   Проведены реальные тестовые запросы.
-   Сделан аргументированный вывод («Go»/«No-Go») на основе полученных данных.

**Ожидаемые результаты:**

-   Заполненный чек-лист тестирования гипотезы.
-   Текст промпта для MVP-агента.
-   Таблица с результатами тестовых запросов.
-   Итоговое решение с кратким обоснованием.

**Тайминг:** 2 часа.

## Задание 3. Разработка метрик и плана Error Handling (Продвинутый уровень)

**Задача:** Для гипотезы, получившей статус «Go», разработать систему контроля и план реагирования на сбои.

**Инструкция:**

1.  **Определите метрики (20 минут):**
    *   Выберите 1-2 ключевые **бизнес-метрики**, которые покажут ценность вашего агента (например, ROI, экономия времени, рост CSAT).
    *   Выберите 1-2 ключевые **операционные метрики**, которые покажут качество его работы (например, Accuracy, Error Rate).
    *   Для каждой метрики укажите, как вы будете ее измерять.
2.  **Создайте план Error Handling (25 минут):**
    *   На основе лекционного материала заполните матрицу «Тип ошибки — Действие» для вашего агента.
    *   Рассмотрите минимум 3 типа ошибок: **галлюцинация, неверная интерпретация, технический сбой**.
    *   Для каждого типа ошибки определите уровень критичности, способ обнаружения, ответственного и краткий протокол действий.

**Критерии оценки:**

-   Выбраны релевантные бизнес- и операционные метрики.
-   Для каждой метрики предложен способ измерения.
-   Матрица Error Handling заполнена для трех обязательных типов ошибок.
-   Протоколы действий являются конкретными и выполнимыми.

**Ожидаемые результаты:**

-   Таблица с описанием 2-4 ключевых метрик для вашего AI-агента.
-   Заполненная матрица Error Handling.

**Тайминг:** 45 минут.
