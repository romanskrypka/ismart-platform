# НЕДЕЛЯ 1, ДЕНЬ 4: Рабочие шаблоны и чек-листы

---

## Введение

Эти шаблоны и фреймворки предназначены для того, чтобы помочь вам систематизировать процесс планирования, внедрения и оценки AI-агентов в вашем бизнесе. Используйте их как отправную точку для своих собственных проектов. Они помогут вам структурировать мысли, не упустить важные детали и принимать более обоснованные решения.

---

### Шаблон 1: Проектирование Промпта для AI-агента

**Цель:** Структурированно создать эффективный промпт для языковой модели. Используйте эту таблицу каждый раз, когда вы проектируете «мозг» нового агента.

| Элемент Промпта | Содержание | 
| :--- | :--- | 
| **1. Роль (Role)** | *Четко определите, кем должен быть AI. Пример: «Ты — старший аналитик отдела продаж...»* | 
| **2. Контекст (Context)** | *Опишите бизнес-ситуацию и задачу. Пример: «Ты анализируешь входящие письма от потенциальных клиентов...»* | 
| **3. Задача (Task)** | *Сформулируйте конкретное действие. Пример: «Твоя задача — проанализировать текст письма и извлечь из него ключевую информацию»* | 
| **4. Формат вывода (Output Format)** | *Опишите точную структуру, в которой AI должен вернуть ответ. Всегда предпочитайте JSON. Пример: «Верни результат строго в формате JSON со следующими полями: `is_lead` (boolean), `product_category` (string), `urgency` (string: low, medium, high)»* | 
| **5. Ограничения и правила (Constraints & Rules)** | *Укажите, чего AI не должен делать. Пример: «Не выдумывай информацию. Если данных в письме нет, оставляй поле пустым. Отвечай только в формате JSON, без лишних слов»* | 
| **6. Примеры (Examples / Few-shot)** | *Приведите 1-3 примера пары «входной текст -> ожидаемый JSON-ответ». Это значительно улучшает точность.* |

### Шаблон 2: Матрица Оценки Бизнес-кейса для AI-агента

**Цель:** Оценить целесообразность внедрения AI-агента для решения конкретной бизнес-проблемы. Заполните эту матрицу перед стартом любого проекта по интеллектуальной автоматизации.

| Критерий | Описание / Вопросы для анализа | Оценка (1-5) | 
| :--- | :--- | :--- | 
| **1. Боль (Pain Point)** | Насколько острой является проблема, которую мы решаем? Приводит ли она к прямым финансовым потерям, потере клиентов, репутационным рискам? | | 
| **2. Частота (Frequency)** | Как часто возникает эта проблема? Это ежедневный, еженедельный или редкий процесс? (Чем чаще, тем выше ценность автоматизации). | | 
| **3. Объем (Volume)** | Каков объем ручной работы, связанной с этим процессом? Сколько человеко-часов в месяц на него уходит? | | 
| **4. Структурированность (Structure)** | Насколько процесс основан на правилах и работе с данными? Можно ли его четко описать? (Хаотичные, творческие процессы хуже поддаются автоматизации). | | 
| **5. Измеримость (Measurability)** | Сможем ли мы четко измерить эффект от внедрения в виде конкретных KPI (экономия, скорость, точность)? | | 
| **Итоговая оценка:** | *Сумма баллов* | | 

**Рекомендация:** Проекты с итоговой оценкой выше 18-20 имеют высокий потенциал успеха и должны рассматриваться в первую очередь.

### Фреймворк 3: Архитектура AI-агента (Solution Architecture)

**Цель:** Спроектировать высокоуровневую архитектуру AI-агента. Используйте эту таблицу для визуализации компонентов и их функций.

| Компонент | Назначение | Выбранный инструмент/сервис | Альтернативы | 
| :--- | :--- | :--- | :--- | 
| **Источник данных** | *Откуда агент получает информацию для работы?* | | | 
| **Триггер (Trigger)** | *Что запускает рабочий процесс агента?* | | | 
| **Оркестратор (Orchestrator)** | *Ядро, управляющее логикой и последовательностью действий.* | | | 
| **Мозг (Brain) / LLM** | *Какой AI будет выполнять интеллектуальную часть работы?* | | | 
| **Инструменты (Tools)** | *С какими внешними системами/API агент должен взаимодействовать для выполнения задачи?* | | | 
| **База знаний (Knowledge Base)** | *Где хранятся данные, необходимые агенту для работы (если применимо)?* | | | 
| **Система назначения (Dispatcher)** | *Куда или кому агент передает результаты своей работы?* | | |

### Чек-лист 4: Внедрение и запуск AI-агента

**Цель:** Пройти все необходимые шаги для успешного запуска AI-агента в продуктивную эксплуатацию. Не переходите к следующему этапу, пока не выполнены все пункты предыдущего.

**Этап 1: Планирование и Дизайн (Planning & Design)**

- [ ] Определена и задокументирована бизнес-проблема.
- [ ] Заполнена Матрица Оценки Бизнес-кейса.
- [ ] Определены и согласованы ключевые метрики успеха (KPI).
- [ ] Спроектирована архитектура решения (заполнен Фреймворк 3).
- [ ] Разработан и протестирован промпт для AI (заполнен Шаблон 1).

**Этап 2: Разработка и Тестирование (Development & Testing)**

- [ ] Настроена среда для разработки (например, аккаунт в n8n).
- [ ] Настроены интеграции со всеми необходимыми системами (API-ключи, доступы).
- [ ] Собран основной рабочий процесс (workflow) агента.
- [ ] Проведено модульное тестирование (каждый компонент работает).
- [ ] Проведено интеграционное тестирование на 10-20 реальных примерах.
- [ ] Реализован механизм Human-in-the-Loop для спорных случаев.

**Этап 3: Внедрение и Онбординг (Deployment & Onboarding)**

- [ ] Подготовлена документация для пользователей и службы поддержки.
- [ ] Проведено обучение для команды, которая будет работать с агентом.
- [ ] Проинформированы все заинтересованные стороны о запуске.
- [ ] Проведен пилотный запуск на ограниченном объеме данных/пользователей.
- [ ] Собрана обратная связь от пилотной группы.

**Этап 4: Эксплуатация и Мониторинг (Operation & Monitoring)**

- [ ] Агент запущен в полную продуктивную эксплуатацию.
- [ ] Настроен дашборд для мониторинга ключевых метрик (KPI).
- [ ] Настроена система оповещений о сбоях в работе агента.
- [ ] Назначен ответственный за поддержку и развитие агента.
- [ ] Регулярно (например, раз в месяц) проводится анализ работы агента и планируются улучшения.

### Матрица 5: Оценка и Митигация Рисков

**Цель:** Проактивно управлять рисками проекта по внедрению AI-агента.

| Категория риска | Пример риска | Вероятность (Н/С/В) | Влияние (Н/С/В) | План митигации (предотвращения) | Ответственный | 
| :--- | :--- | :--- | :--- | :--- | :--- | 
| **Технологический** | *Неточность AI-модели (галлюцинации, неверная классификация)* | | | *1. Использовать более продвинутую модель (GPT-4 вместо GPT-3.5). 2. Внедрить механизм Human-in-the-Loop для запросов с низкой уверенностью. 3. Регулярный аудит точности.* | | 
| **Операционный** | *Сбой в работе API одного из компонентов (например, n8n или OpenAI)* | | | *1. Настроить систему мониторинга и алертинга. 2. Разработать инструкцию для поддержки по перезапуску процесса.* | | 
| **Безопасность** | *Утечка конфиденциальных данных через внешний API* | | | *1. Использовать API, которые не сохраняют данные. 2. Внедрить процесс анонимизации данных перед отправкой. 3. Для критичных данных — использовать self-hosted модели.* | | 
| **Человеческий фактор** | *Сопротивление команды внедрению, саботаж* | | | *1. Провести воркшоп для команды с объяснением ценности агента. 2. Вовлечь ключевых сотрудников в процесс тестирования. 3. Сместить акцент с «замены» на «помощь».* | | 
| **Бизнес-риск** | *Затраты на внедрение не окупаются* | | | *1. Начинать с MVP (минимально жизнеспособного продукта). 2. Постоянно отслеживать KPI и корректировать работу агента. 3. Декомпозировать проект на фазы с оценкой ROI после каждой.* | |
