# Неделя 1, День 1: Рабочие шаблоны и чек-листы

## Введение

Хорошие инструменты и структурированный подход — залог успеха в любом сложном деле. Этот раздел предоставляет вам готовые шаблоны, чек-листы и фреймворки, которые помогут вам систематизировать процесс проектирования и оценки AI-агентов. Используйте их как отправную точку для своих собственных проектов, адаптируя под конкретные задачи.

## Шаблон 1: Проектирование AI-агента

**Цель:** Структурированно описать архитектуру и компоненты будущего AI-агента перед началом разработки. Заполнение этой таблицы поможет вам четко сформулировать требования и избежать ошибок на этапе реализации.

**Инструкция:** Скопируйте эту таблицу и заполните ее для вашего собственного проекта AI-агента.

| Компонент | Описание | Пример: Агент "Финансовый аналитик" |
| :--- | :--- | :--- |
| **Название агента** | Краткое, запоминающееся имя. | `FinanceBot` |
| **Основная цель** | Какую главную проблему решает агент? Опишите в одном предложении. | `Автоматический сбор и анализ квартальной отчетности публичных компаний.` |
| **Ключевые задачи (Goals)** | 3-5 основных задач, которые должен выполнять агент для достижения цели. | `1. Находить последнюю отчетность (10-Q/10-K). 2. Извлекать ключевые метрики (выручка, прибыль). 3. Сравнивать с прошлым годом. 4. Генерировать сводный отчет.` |
| **Триггер (Запуск)** | Какое событие или команда запускает работу агента? | `Команда пользователя: "Проанализируй отчетность компании X за Q3 2026".` |
| **Необходимые инструменты (Tools)** | Перечислите API, библиотеки, функции, которые понадобятся агенту. | `- Поиск Google/Bing API (для нахождения отчетов).<br>- Библиотека BeautifulSoup (для парсинга HTML/XML).<br>- API финансового провайдера (для получения котировок).<br>- Внутренний скилл `send_notification` (для отправки отчета).` |
| **Краткосрочная память** | Какую информацию нужно хранить в рамках одной сессии? | `- URL найденного отчета.<br>- Извлеченные финансовые данные.<br>- Промежуточные выводы.<br>- История команд пользователя.` |
| **Долгосрочная память** | Какую информацию агент должен запоминать между сессиями? | `- Предпочтения пользователя по формату отчета.<br>- Список уже проанализированных компаний.<br>- API-ключи для доступа к сервисам.` |
| **Критерии успеха** | Как вы поймете, что агент успешно справился с задачей? | `Пользователь получил на почту корректный, структурированный отчет с ключевыми метриками и выводами в течение 5 минут после запроса.` |
| **Возможные ошибки и пути их решения** | Что может пойти не так и как агент должен на это реагировать? | `- Отчет не найден -> Сообщить пользователю.<br>- Ошибка парсинга сайта -> Попробовать альтернативный источник.<br>- Невалидный API-ключ -> Уведомить администратора.` |

| :-: | :--- | :---: | :--- |

## Фреймворк 3: Цикл работы AI-агента

**Цель:** Понять и применять на практике фундаментальный цикл принятия решений, который лежит в основе работы большинства автономных систем. Этот фреймворк поможет вам структурировать логику вашего агента.

Часто для описания цикла работы агента используют модель **Observe -> Plan -> Act (Наблюдение -> Планирование -> Действие)**, которая является упрощением классического военного цикла **OODA Loop (Observe-Orient-Decide-Act)**.

| Этап | Описание | Пример: Агент-рекрутер |
| :--- | :--- | :--- |
| **1. Observe (Наблюдение)** | Агент собирает информацию из окружающей среды. Это может быть новый запрос от пользователя, данные с датчика, изменение в базе данных, новая статья в новостной ленте. | `Получена новая команда: "Найди 3 Python-разработчиков с опытом в Django".` |
| **2. Orient (Ориентация/Анализ)** | Это самый важный этап. Агент анализирует полученную информацию в контексте своих знаний, целей и предыдущего опыта. Он «осмысляет» ситуацию. Здесь происходит основная работа LLM по анализу и синтезу. | `Агент анализирует запрос. "Python-разработчик" и "Django" - ключевые навыки. "3" - количество. Он обращается к своей долгосрочной памяти и вспоминает, какие сайты для поиска лучше всего подходят.` |
| **3. Decide (Решение/Планирование)** | На основе анализа агент формирует план действий. Он решает, что именно нужно сделать для достижения цели, какие инструменты использовать и в какой последовательности. | `Агент решает: 1. Использовать инструмент "Поиск по LinkedIn". 2. Сформировать поисковый запрос. 3. Проанализировать топ-10 результатов. 4. Выбрать 3 наиболее релевантных. 5. Сформировать отчет для пользователя.` |
| **4. Act (Действие)** | Агент выполняет первый шаг своего плана, используя выбранный инструмент. Результат этого действия становится новым «наблюдением», и цикл начинается заново. | `Агент выполняет первый шаг: вызывает API LinkedIn с запросом "Python developer Django". Получает список из 10 профилей. Этот список становится новым "наблюдением" для следующего цикла.` |

Этот цикл повторяется до тех пор, пока конечная цель не будет достигнута.

## Матрица 4: Оценка выбора между Чат-ботом и AI-агентом

**Цель:** Помочь принять взвешенное решение о том, какая технология лучше подходит для конкретной бизнес-задачи. Оцените вашу задачу по приведенным ниже критериям.

**Инструкция:** Для каждой характеристики поставьте балл от 1 до 5, где 1 — больше подходит Чат-бот, 5 — больше подходит AI-агент.

| Характеристика задачи | 1 (Чат-бот) | 2 | 3 | 4 | 5 (AI-агент) | Ваша оценка | Обоснование |
| :--- | :--- | :--- | :--- | :--- | :--- | :---: | :--- |
| **Сложность процесса** | Простой, линейный сценарий | | | | Сложный, многошаговый процесс | | `Пример: Процесс требует взаимодействия с 3+ системами.` |
| **Необходимость интеграции** | Не требует или 1 простая интеграция | | | | Требует множества интеграций с API | | `Пример: Нужно получать данные из CRM и отправлять в бухгалтерию.` |
| **Уровень неопределенности** | Все шаги заранее известны | | | | Высокая вариативность, возможны исключения | | `Пример: Клиент может задавать нестандартные вопросы.` |
| **Требуемая автономность** | Требуется постоянный контроль | | | | Полностью автономное выполнение | | `Пример: Задача должна выполняться ночью без участия человека.` |
| **Работа с данными** | Простое извлечение информации | | | | Анализ, синтез и генерация новых данных | | `Пример: Нужно не просто показать цифру, а написать аналитический отчет.` |
| **Персонализация** | Общие ответы для всех | | | | Глубокая персонализация на основе истории | | `Пример: Ответ должен учитывать предыдущие покупки клиента.` |

**Итог:**
*   **Сумма баллов 6-12:** Для вашей задачи, скорее всего, будет достаточно хорошо настроенного чат-бота.
*   **Сумма баллов 13-18:** Можно рассмотреть гибридный подход или простого агента.
*   **Сумма баллов 19-30:** Ваша задача — идеальный кандидат для реализации с помощью полноценного AI-агента.
