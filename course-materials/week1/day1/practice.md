## Задание 1: Настройка AI-ориентированной среды разработки (Cursor)

**Цель:** Установить и настроить интегрированную среду разработки (IDE) Cursor, которая изначально создана для работы с AI, и опробовать ее базовые возможности.

**Уровень сложности:** Базовый

**Тайминг:** 30 минут

### Пошаговые инструкции:

1.  **Скачивание и установка:**
    *   Перейдите на официальный сайт [Cursor](https://cursor.sh/).
    *   Скачайте версию, совместимую с вашей операционной системой (Windows, macOS, Linux).
    *   Установите приложение, следуя инструкциям инсталлятора.

2.  **Первоначальная настройка:**
    *   Запустите Cursor. При первом запуске вам будет предложено настроить AI-функции.
    *   Cursor предлагает несколько вариантов использования AI: через вход в свою учетную запись, с помощью API-ключа от OpenAI или других провайдеров. Для максимальной гибкости выберем вариант с API-ключом.
    *   В настройках найдите раздел «AI» или «LLM» и вставьте свой API-ключ от OpenAI. Если у вас его нет, вам потребуется создать его в личном кабинете на сайте OpenAI.

3.  **Тестирование AI-функций:**
    *   **Chat (Чат с кодом):** Откройте любой свой проект или создайте новый пустой файл (например, `test.py`). Нажмите `Ctrl+K` (или `Cmd+K` на macOS), чтобы открыть чат с AI. Попросите его написать простую функцию, например: `«напиши функцию на Python, которая вычисляет факториал числа»`. Оцените, как быстро и корректно он сгенерирует код.
    *   **Edit (Редактирование кода):** Выделите сгенерированную функцию. Нажмите `Ctrl+J` (или `Cmd+J` на macOS), чтобы войти в режим редактирования. Попросите AI внести изменения, например: `«добавь обработку ошибок для отрицательных чисел»`. Посмотрите, как Cursor изменит существующий код.
    *   **Code Generation (Генерация с нуля):** В пустом файле попросите Cursor сгенерировать целый класс. Например: `«создай класс `Rectangle` с методами для вычисления площади и периметра»`.

### Критерии оценки:

*   Приложение Cursor успешно установлено и запускается.
*   API-ключ OpenAI добавлен в настройки, и AI-функции активны.
*   Вы успешно сгенерировали код с помощью функции «Chat» (`Ctrl+K`).
*   Вы успешно отредактировали существующий код с помощью функции «Edit» (`Ctrl+J`).

### Ожидаемые результаты:

По итогам задания у вас будет полностью настроенная и готовая к работе AI-ориентированная среда разработки. Вы освоите базовые команды для взаимодействия с AI, что значительно ускорит вашу дальнейшую работу над созданием агентов.

## Задание 2: Подключение к MCP-серверу

**Цель:** Научиться подключаться к внешним источникам знаний с помощью MCP (Model Context Protocol). Вы будете использовать публичный MCP-сервер для получения данных, имитируя обращение вашего будущего агента к внешней базе данных.

**Уровень сложности:** Продвинутый

**Тайминг:** 45 минут

### Пошаговые инструкции:

1.  **Установка библиотеки:**
    *   Для взаимодействия с MCP-серверами мы будем использовать библиотеку `fastmcp`. Откройте терминал или командную строку и установите ее:
        ```bash
        pip install fastmcp
        ```

2.  **Изучение концепции:**
    *   MCP-сервер — это, по сути, API, которое предоставляет AI-модели контекст (данные) в стандартизированном виде. Агент отправляет запрос, а сервер возвращает релевантную информацию. Мы будем использовать демонстрационный сервер, который отдает информацию о вымышленных продуктах.

3.  **Написание скрипта для подключения:**
    *   Создайте новый Python-файл, например, `mcp_client.py`.
    *   Импортируйте необходимые классы и напишите код для подключения к публичному MCP-серверу и получения данных.

    ```python
    import asyncio
    from fastmcp import MCPClient

    # URL публичного демо-сервера
    MCP_SERVER_URL = "https://demo.mcp.ai/"

    async def get_product_info(product_name: str):
        """Асинхронная функция для получения информации о продукте с MCP-сервера."""
        print(f"Подключаемся к MCP-серверу по адресу: {MCP_SERVER_URL}")
        async with MCPClient(MCP_SERVER_URL) as client:
            print(f"Отправляем запрос для получения информации о продукте: '{product_name}'")
            # 'get_product_details' - это название "скилла" или "ручки" на стороне сервера
            response = await client.request("get_product_details", {"product_name": product_name})
            
            print("\n--- Ответ от MCP-сервера ---")
            if response.status == 200:
                print(f"Статус: {response.status}")
                print("Данные:")
                # Выводим полученные данные в удобном виде
                for key, value in response.data.items():
                    print(f"  {key}: {value}")
            else:
                print(f"Ошибка! Статус: {response.status}")
                print(f"Сообщение: {response.error}")
            print("---------------------------")

    async def main():
        # Попробуем получить информацию о двух разных продуктах
        await get_product_info("AI-пылесос 'Чистобот-9000'")
        print("\n")
        await get_product_info("Квантовый компьютер 'Квантум-1'")

    if __name__ == "__main__":
        # Запускаем асинхронный код
        asyncio.run(main())

    ```

4.  **Запуск и анализ:**
    *   Запустите скрипт из вашего терминала:
        ```bash
        python mcp_client.py
        ```
    *   Проанализируйте вывод. Вы должны увидеть статус ответа (200 для успеха) и данные, полученные от сервера. Обратите внимание, как структурирован ответ.

### Критерии оценки:

*   Библиотека `fastmcp` успешно установлена.
*   Скрипт `mcp_client.py` создан и не содержит синтаксических ошибок.
*   При запуске скрипт успешно подключается к MCP-серверу и получает ответ со статусом 200.
*   В консоль выводится информация о запрошенных продуктах.

### Ожидаемые результаты:

Вы получите работающий пример кода, который демонстрирует, как AI-агент может запрашивать информацию из внешних, структурированных источников данных. Это ключевой навык для создания агентов, способных оперировать актуальной и специфичной для бизнеса информацией.

## Задание 3: Разработка базового набора «скиллов»

**Цель:** Создать три простых, но функциональных «скилла» (навыка) для вашего будущего агента. Скилл — это, по сути, инструмент, который агент может использовать для выполнения определенного действия.

**Уровень сложности:** Продвинутый

**Тайминг:** 60 минут

### Пошаговые инструкции:

1.  **Создание файла для скиллов:**
    *   Создайте новый Python-файл с названием `skills.py`. В нем вы будете определять все ваши навыки в виде отдельных функций.

2.  **Скилл 1: Коммуникация (Отправка уведомления)**
    *   Напишите функцию `send_notification`. Поскольку настройка реальной отправки email требует дополнительных шагов, мы сымитируем это действие, просто выводя сообщение в консоль. Главное — правильно оформить функцию и ее описание (docstring), чтобы агент «понял», что она делает.

    ```python
    def send_notification(recipient: str, message: str) -> str:
        """
        Отправляет уведомление указанному получателю.
        Имитирует отправку email или сообщения в мессенджер.

        :param recipient: Адресат сообщения (например, 'user@example.com' или 'team_lead').
        :param message: Текст сообщения.
        :return: Строка, подтверждающая успешную отправку.
        """
        print(f"[УВЕДОМЛЕНИЕ] Отправка сообщения для {recipient}: '{message}'")
        return f"Сообщение для {recipient} успешно отправлено."

    ```

3.  **Скилл 2: Аналитика (Простое вычисление)**
    *   Создайте функцию `calculate_average`. Этот скилл будет принимать список чисел и возвращать их среднее значение. Это пример инструмента для выполнения математических операций.

    ```python
    from typing import List

    def calculate_average(numbers: List[float]) -> float:
        """
        Вычисляет среднее арифметическое для списка чисел.

        :param numbers: Список чисел для вычисления среднего.
        :return: Среднее значение. Если список пуст, возвращает 0.
        """
        if not numbers:
            return 0.0
        return sum(numbers) / len(numbers)

    ```

4.  **Скилл 3: Управление задачами (Добавление в список дел)**
    *   Напишите функцию `add_task_to_list`. Она будет имитировать добавление задачи в таск-менеджер. Для простоты мы будем использовать обычный глобальный список Python в качестве нашего «таск-менеджера».

    ```python
    # Глобальный список для имитации базы данных задач
    TODO_LIST = []

    def add_task_to_list(task_description: str) -> str:
        """
        Добавляет новую задачу в глобальный список дел (TODO_LIST).

        :param task_description: Описание задачи.
        :return: Строка, подтверждающая добавление задачи и текущее количество задач.
        """
        print(f"[ЗАДАЧА] Добавление задачи: '{task_description}'")
        TODO_LIST.append(task_description)
        return f"Задача '{task_description}' добавлена. Всего задач в списке: {len(TODO_LIST)}."

    ```

5.  **Тестирование скиллов:**
    *   В конце файла `skills.py` добавьте блок для проверки работоспособности ваших функций.

    ```python
    if __name__ == "__main__":
        print("--- Тестирование скиллов ---")
        
        # Тест скилла коммуникации
        print("\n1. Тест 'send_notification':")
        result1 = send_notification("boss@mycompany.com", "Отчет за квартал готов.")
        print(f"Результат вызова: {result1}")

        # Тест скилла аналитики
        print("\n2. Тест 'calculate_average':")
        my_numbers = [10.5, 20.0, 15.5, 5.0]
        result2 = calculate_average(my_numbers)
        print(f"Среднее для {my_numbers} = {result2}")

        # Тест скилла управления задачами
        print("\n3. Тест 'add_task_to_list':")
        result3_1 = add_task_to_list("Подготовить презентацию для встречи в понедельник.")
        print(f"Результат вызова: {result3_1}")
        result3_2 = add_task_to_list("Купить билеты на конференцию.")
        print(f"Результат вызова: {result3_2}")
        print(f"Итоговый список задач: {TODO_LIST}")
        
        print("\n--- Тестирование завершено ---")
    ```

6.  **Запуск и проверка:**
    *   Запустите файл `skills.py` и убедитесь, что все тестовые вызовы проходят корректно и вывод в консоли соответствует ожиданиям.

### Критерии оценки:

*   Создан файл `skills.py`.
*   Реализованы все три функции: `send_notification`, `calculate_average`, `add_task_to_list`.
*   Каждая функция имеет корректную аннотацию типов и подробный docstring, объясняющий ее назначение, параметры и возвращаемое значение.
*   Тестовый блок `if __name__ == "__main__":` присутствует и корректно проверяет работу всех трех скиллов.
*   При запуске файла `skills.py` в консоль выводится осмысленный результат работы каждой функции.

### Ожидаемые результаты:

Вы создадите свой первый набор инструментов (скиллов), который в дальнейшем сможет использовать ваш AI-агент. Вы поймете, как важно правильно описывать функции (docstrings), чтобы языковая модель могла понять, для чего нужен тот или иной инструмент и как его использовать. Это заложит основу для создания по-настояшему автономного и полезного агента.
