## Рабочие шаблоны и чек-листы

### 1. Шаблон для описания промпта в библиотеке

Используйте эту таблицу в Markdown для добавления нового промпта в общую библиотеку. Это обеспечит единообразие и поможет коллегам быстро понять, для чего нужен промпт и как его использовать.

```markdown
### Название промпта: [Краткое и понятное название]

| Категория | Описание |
| :--- | :--- |
| **ID Промпта** | `[Уникальный идентификатор, например, MKT-001]` |
| **Автор** | `[Ваше Имя]` |
| **Дата создания** | `[ДД.ММ.ГГГГ]` |
| **Версия** | `1.0` |
| **Задача** | `[Что должен сделать AI? Опишите цель как можно точнее]` |
| **Контекст (Роль)** | `[Какую роль должен играть AI? Например: "Ты - опытный финансовый аналитик"]` |
| **Аудитория** | `[Для кого предназначен результат? Например: "Для совета директоров"]` |
| **Тон** | `[В каком стиле должен быть ответ? Например: Официальный, Креативный, Дружелюбный]` |
| **Ограничения** | `[Чего AI не должен делать? Например: "Не использовать жаргон", "Не ссылаться на внешние источники"]` |
| **Формат вывода** | `[В каком виде предоставить результат? Например: Таблица, Маркированный список, JSON]` |
| **Теги** | `[#тег1, #тег2, #анализ, #маркетинг]` |

**Текст промпта:**

```
[Вставьте сюда полный текст промпта]
```

**Пример использования:**

```
[Приведите конкретный пример, как можно использовать этот промпт, заполнив переменные]
```

**Ожидаемый результат:**

```
[Опишите или вставьте пример того, что должно получиться в итоге]
```

```

### 2. Чек-лист для AI Champion'а

Этот чек-лист поможет AI Champion'у структурировать свою работу и не забыть о ключевых обязанностях.

**Ежедневные задачи:**

- [ ] Проверить новые запросы на консультацию от коллег.
- [ ] Ответить на вопросы в командном чате по AI.

**Еженедельные задачи:**

- [ ] Провести ревью новых промптов, предложенных для добавления в библиотеку.
- [ ] Добавить 1-2 новых утвержденных промпта в библиотеку.
- [ ] Проверить и актуализировать 1-2 раздела в Shared Knowledge Base.
- [ ] Провести 30-минутную встречу "AI-кофе" для обмена опытом и новостями с командой.
- [ ] Изучить одну новую статью или кейс по применению AI в вашей сфере.

**Ежемесячные задачи:**

- [ ] Подготовить краткий отчет об использовании AI в команде (количество решенных задач, самые популярные промпты, экономия времени).
- [ ] Провести воркшоп или мини-тренинг по новой функции или инструменту AI.
- [ ] Собрать обратную связь от команды по работе с AI и составить список идей для улучшения.
- [ ] Провести аудит библиотеки промптов и SKB на предмет устаревшей информации.

**Ежеквартальные задачи:**

- [ ] Провести встречу с руководством для обсуждения статуса внедрения AI и планов на следующий квартал.
- [ ] Проанализировать метрики эффективности и скорректировать стратегию использования AI.
- [ ] Организовать большое командное мероприятие (хакатон, мозговой штурм) для генерации новых идей по применению AI.

### 3. Фреймворк для разработки командного протокола взаимодействия с AI

Используйте этот фреймворк как основу для создания собственного протокола. Заполните каждый раздел в соответствии с целями и процессами вашей команды.

| Раздел | Вопросы для заполнения | Ваш ответ |
| :--- | :--- | :--- |
| **1. Цели и Принципы** | - Для чего наша команда использует AI? <br>- Каковы наши главные принципы при работе с AI (например, этичность, ответственность, конфиденциальность)? | |
| **2. Инструменты** | - Какие конкретные AI-модели, платформы и агенты разрешены к использованию? <br>- Где находится наша командная библиотека промптов? <br>- Где находится наша Shared Knowledge Base? | |
| **3. Процессы** | - Как выглядит стандартный воркфлоу "задача -> AI -> результат -> проверка"? <br>- Каков процесс добавления нового промпта в библиотеку (кто предлагает, кто утверждает)? <br>- Кто и как часто обновляет Shared Knowledge Base? | |
| **4. Роли** | - Кто является нашим AI Champion'ом? <br>- Каковы его/ее основные обязанности и к кому обращаться за помощью? | |
| **5. Безопасность** | - Какие типы данных категорически запрещено передавать во внешние AI-сервисы? <br>- Какова наша политика в отношении проверки фактов и цитирования источников при использовании AI? | |
| **6. Оценка** | - Какие метрики мы используем для оценки эффективности AI (например, сэкономленное время, количество выполненных задач)? <br>- Как и как часто мы собираем обратную связь от команды? | |
| **7. Развитие** | - Как мы будем обучать новых сотрудников работе с AI? <br>- Как мы будем поощрять инновации и поиск новых способов применения AI? | |
